Issue #3

툴 desk는 브랜드 소개가 아니라 변화 패턴을 쌓는 아카이브여야 한다

OpenAI, Anthropic, Cursor, Vercel, Supabase 다섯 desk를 먼저 엽니다. 각 desk는 공식 채널, 관련 카테고리, current seed, 다음 큐를 묶는 탐색 축입니다.

OpenAI Anthropic Cursor Vercel Supabase

Desk rule

  • Desk는 공식 채널과 실무 패턴을 함께 적립합니다.
  • 아직 글 수가 적으므로 회사 소개보다 seed와 queue를 먼저 노출합니다.
  • 카테고리와 desk는 서로 왕복 가능한 링크 구조를 유지합니다.

Desk 01

OpenAI

Breaking Security

모델 lifecycle, API changelog, release notes, deprecations가 모두 여기로 모입니다. 현재 두 seed 모두 OpenAI 공식 근거를 포함합니다.

Seed 01: deprecations operations brief Seed 02: endpoint / rollout engineering brief Category: Model / API changes

Desk 02

Anthropic

Security Education

아직 seed는 없지만, policy, model updates, enterprise workflow changes가 OpenAI와 비교될 수 있는 다음 축입니다.

Category: Model / API changes Category: Automation / agent features

Desk 03

Cursor

Security Deployment

인터뷰 기준으로 rules file, extension version, team rollout note가 한 세트로 움직이는 desk입니다. 다음 seed 후보가 여기에서 나옵니다.

Category: Coding tool updates Category: Automation / agent features

Desk 04

Vercel

Deployment Cost

region, runtime, build, edge, quota처럼 deploy와 billing이 겹치는 변화를 추적합니다. Seed 02에서 OpenAI API 변화와 같이 읽히는 desk입니다.

Seed 02: rollout / fallback sequence Category: Deploy / infrastructure changes Category: Pricing / plan changes

Desk 05

Supabase

Deployment Non-breaking

database, edge functions, auth, pricing and branch workflow changes를 장기적으로 담을 desk입니다. 현재는 infra / pricing 카테고리의 follow-up 대상으로 둡니다.

Category: Deploy / infrastructure changes Category: Pricing / plan changes
